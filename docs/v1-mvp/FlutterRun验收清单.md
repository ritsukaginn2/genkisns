> 日期：2026-05-24

---

## 一、验收原则

Phase 4 之后不再用 `board.html` 或 `build/web` 作为验收入口。

- `board.html`：只作为 Phase 3 设计板归档，用来看页面覆盖和历史设计状态。
- `build/web`：只作为开发调试备用，不作为 Phase 4+ 验收。
- `flutter run`：Phase 4+ 的正式验收方式。

验收时应运行在 iOS Simulator、Android Emulator 或真实移动设备上。没有可用设备时，应先解决设备环境，不用 Web 替代。

---

## 二、运行入口

### 2.1 查看设备

```bash
cd apps/mobile
flutter devices
```

### 2.2 运行到 iOS 真机或模拟器

```bash
cd apps/mobile
flutter run -d <device-id>
```

需要只安装并启动、不常驻终端时：

```bash
cd apps/mobile
flutter run -d <device-id> --no-resident
```

### 2.3 当前机器状态

- Xcode / iOS 工具链可用。
- Android SDK 当前不可用，因此 Android Emulator 暂不作为本机验收目标。
- 如果无线 iPhone / iPad 连接失败，需要解锁设备、确认开发者模式、或改用数据线连接。

## 三、Phase 4 验收范围

Phase 4 验收的目标是确认：当前已签过的 Flutter UI 和交互，可以脱离 Web 设计板，在真实 Flutter runtime 中运行。

### 3.1 启动与首页

- [ ] App 通过 `flutter run` 启动成功。
- [ ] 启动后直接进入首页。
- [ ] 首屏没有 onboarding 或配置流程。
- [ ] 首页没有底部 Tab。
- [ ] 空状态展示清楚。
- [ ] 页面安全区、顶部区域、底部区域在设备上正常。

### 3.2 发布笔记

- [ ] 点击首页发布按钮进入发布笔记页。
- [ ] 从首页进入发布页时，文字输入框自动聚焦并弹出键盘。
- [ ] 文字、图片和视频都为空时不能发布。
- [ ] 输入文字后可以发布。
- [ ] 不输入文字、只添加图片也可以发布。
- [ ] 不输入文字、只添加视频也可以发布。
- [ ] 点击添加媒体后出现媒体来源选择层：相机、相册、取消。
- [ ] 选择相机后进入系统相机。
- [ ] 在系统相机拍照后添加 1 张图片。
- [ ] 在系统相机拍视频后添加 1 个视频。
- [ ] 选择相册后进入 App 内相册选择层，并可以多选图片。
- [ ] 相册选择层可以选择 1 个视频。
- [ ] 再次打开相册选择层时，已添加图片保持勾选。
- [ ] 取消勾选已选图片后，发布页图片同步移除。
- [ ] 每个已添加媒体右上角都有删除按钮。
- [ ] 删除按钮位置在媒体右上角并略微超出媒体边界。
- [ ] 图片最多 9 张，满图后不能继续添加。
- [ ] 视频最多 1 个，且不能和图片混排。
- [ ] 图片或视频发布后复制到 App 本地目录。
- [ ] 发布成功后直接回首页。
- [ ] 发布成功后不出现“AI 将会评论”的提示弹窗。

### 3.3 首页发布结果

- [ ] 首页出现刚发布的笔记。
- [ ] 新笔记排在最前。
- [ ] 笔记卡片展示文字摘要。
- [ ] 有图片时展示图片区域。
- [ ] 有视频时展示视频封面、播放标识和时长。
- [ ] 笔记卡片展示点赞数和评论数。

### 3.4 笔记详情互动

- [ ] 点击首页笔记卡片进入笔记详情页。
- [ ] 详情页展示正文、媒体、发布时间、AI 点赞数、AI 评论。
- [ ] 视频笔记详情页可以打开视频播放器。
- [ ] 点击帖子喜欢后，按钮变为已喜欢，喜欢数 +1。
- [ ] 再次点击帖子喜欢后，按钮恢复未喜欢，喜欢数 -1。
- [ ] 点击评论喜欢后，该评论喜欢状态切换，喜欢数 +1。
- [ ] 再次点击评论喜欢后，该评论喜欢状态恢复，喜欢数 -1。
- [ ] 点击评论回复后，底部出现回复输入层。
- [ ] 输入回复并发送后，回复输入层关闭。
- [ ] 发送后的本地二级回复展示在目标评论下方。
- [ ] 评论数随本地回复 +1。
- [ ] 自己发送的本地回复可以触发删除。
- [ ] 删除前出现确认层。
- [ ] 确认删除后，该回复从评论下方移除。
- [ ] 评论数随删除回复 -1。

### 3.5 我的与子页面

- [ ] 点击首页头像进入我的页。
- [ ] 我的页展示用户资料摘要。
- [ ] 我的页可以进入 AI 好友页。
- [ ] AI 好友页展示至少 5 个 AI 好友。
- [ ] AI 好友具备可区分的名字、头像、关系、性格和说话风格。
- [ ] 我的页可以进入关于 GenkiSNS 页。
- [ ] 关于页说明 AI 互动由本地模板生成。
- [ ] 关于页说明 V1 不会把发帖文字发送到云端 LLM。
- [ ] 关于页说明图片和视频在 V1 中只保存在本机。
- [ ] 我的页可以进入 UI 实验室。
- [ ] UI 实验室只读展示 A/B/C，不影响正式 App 路由。

## 四、Phase 5 数据层验收范围

- [ ] 发布笔记后通过 `PostRepository` 写入 SQLite 本地数据库。
- [ ] 纯文字、纯图片、图文、纯视频、文字 + 视频笔记都可以创建。
- [ ] 空笔记被拒绝。
- [ ] 本地互动生成失败时使用 fallback，帖子不丢失。
- [ ] 帖子喜欢、评论喜欢、本地回复、删除回复写入本地数据库。
- [ ] App 重启后，已发布笔记、图片、视频、AI 评论、喜欢状态和本地回复可以恢复。
- [ ] 相册选择层再次打开时，已选相册图片保持勾选。
- [ ] 媒体引用保留 `type`、`source`、`localRef`、`thumbnailRef`、`durationMillis`、`sortIndex`。
- [ ] SQLite 表覆盖 `posts`、`post_images`、`comments`、`local_replies`。

## 五、当前不验收的内容

- [ ] 云端账号与跨设备同步。
- [ ] 真实 LLM 后端、队列、额度和付费。

这些进入后续版本实现。

---

## 六、通过标准

Phase 5 通过条件：

- App 可以通过 `flutter run` 在目标设备或模拟器启动。
- V1 主流程可以在 Flutter runtime 中走通。
- V1 没有底部 Tab、onboarding、配置流程。
- 没有明显白屏、错位、重叠、返回按钮不一致。
- 本地 SQLite 数据层、本地互动 fallback、喜欢、回复和媒体引用流程可以稳定工作。
- 重启 App 后，本机发布内容和互动状态仍可恢复。
