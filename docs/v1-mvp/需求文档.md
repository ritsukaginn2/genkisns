# GenkiSNS V1 需求文档

> 版本：v1.0（核心闭环）
> 日期：2026-05-24
> 状态：V1 范围已确认
> 上级文档：[../需求文档.md](../需求文档.md)

---

## 一、版本目标

V1 只验证一个核心体验：用户发布一条笔记后，可以在笔记详情页看到 AI 好友的点赞和评论，从而获得“被看见”的感觉。

V1 是本地单人原型，用户进入 App 后直接使用。

---

## 二、功能范围

### 2.1 启动与默认资料

- 用户打开 App 后直接进入首页。
- 使用开发阶段内置的默认用户资料。
- 使用开发阶段内置的默认 AI 好友列表。
- V1 不提供创建资料、选择好友或配置流程。
- IP 属地不在设置阶段填写，发布时像其他 SNS 一样根据网络自动识别。

### 2.2 AI 好友

- 提供至少 5 个预设 AI 好友。
- 每个 AI 好友包含名字、头像、关系、性格关键词、说话风格。
- AI 好友用于生成点赞和评论。

### 2.3 发布笔记

- 支持发布纯文字笔记。
- 支持发布纯图片笔记。
- 支持发布文字 + 图片笔记，图片最多 9 张。
- 点击添加图片后，底部出现图片来源选择层：拍照、从相册选择、取消。
- 拍照一次添加 1 张图片。
- 从相册选择支持多选，最多补足到 9 张。
- 再次打开相册时，已添加的相册图片保持勾选；确认后以当前勾选集合同步发布页图片。
- 文字和图片二选一必填，不能发布空笔记。
- 从首页进入发布页时，文字输入框自动聚焦。
- 发布成功后回到首页，用户可从首页打开笔记详情。

### 2.4 首页

- 首页展示用户自己发布的笔记。
- 笔记按发布时间倒序排列。
- 笔记卡片展示图片、文字摘要、点赞数、评论数。

### 2.5 笔记详情

- 根据笔记内容展示正文和/或图片、发布时间。
- 展示 AI 点赞数。
- 展示 AI 评论列表。
- 发布后 10 秒内出现至少 1 条 AI 评论。
- 点击帖子喜欢按钮后，喜欢数和按钮视觉状态在当前页原地切换。
- 点击评论喜欢按钮后，该评论的喜欢数和按钮视觉状态在当前页原地切换。
- 点击评论回复后，底部弹出回复输入层；发送后在目标评论下方展示本地二级回复。
- 自己发送的本地二级回复可以删除，删除前需要确认。
- V1 的评论回复只完成本地 UI 状态，不要求持久化或触发 LLM。

### 2.6 LLM 调用

- 使用开发者预配置的 OpenAI 兼容接口或代理服务。
- App 内不暴露供应商配置 UI。
- App 内不保存或展示 LLM 供应商密钥。
- LLM 返回结构化 JSON，业务层解析后写入 V1 会话内数据。
- LLM 调用失败时，帖子仍发布成功，并使用备用模板生成少量评论。

### 2.7 数据生命周期

- V1 使用单次 App 会话内数据。
- 用户在本次运行中发布的笔记、图片引用、AI 评论、喜欢状态和本地回复需要可用。
- App 重启后不要求恢复 V1 会话数据。

### 2.8 关于页

- 说明 AI 互动由 LLM 生成。
- 说明发帖文字会发送给开发者预配置的 LLM 供应商。
- 说明图片在 V1 中不发送给 LLM 供应商。

## 三、页面清单

| 页面 | 内容 |
|------|------|
| 首页 | 用户笔记列表 |
| 发布笔记页 | 文字输入、图片添加、发布按钮 |
| 笔记详情页 | 笔记内容、点赞数、AI 评论 |
| 我的页 | 用户资料摘要、AI 好友入口、UI 实验室入口、关于入口 |
| AI 好友页 | 展示 V1 内置的 AI 好友 |
| UI 实验室 | A/B/C 设计方向只读展示 |
| 关于页 | AI 生成说明、数据发送说明 |

---

## 四、导航结构

- V1 不使用底部导航。
- 首页是根页面。
- 首页点击发布按钮进入发布笔记页。
- 首页点击头像进入我的页。
- 首页点击笔记卡片进入笔记详情页。
- 发布成功后返回首页。
- 我的页进入 AI 好友页、UI 实验室、关于页。
- 子页面使用返回按钮回到上一页。

---

## 五、详情互动状态

| 触发 | UI 结果 |
|------|---------|
| 点帖子喜欢 | 帖子喜欢按钮变为已喜欢，喜欢数 +1 |
| 再点帖子喜欢 | 帖子喜欢按钮恢复未喜欢，喜欢数 -1 |
| 点评论喜欢 | 该评论喜欢按钮变为已喜欢，评论喜欢数 +1 |
| 再点评论喜欢 | 该评论喜欢按钮恢复未喜欢，评论喜欢数 -1 |
| 点评论回复 | 底部出现回复输入层 |
| 点回复输入层关闭 | 回复输入层关闭 |
| 点回复输入层发送 | 回复输入层关闭，目标评论下方出现本地二级回复，评论数 +1 |
| 点自己回复的删除按钮 | 底部出现删除确认层 |
| 确认删除自己回复 | 删除该二级回复，评论数 -1 |

## 七、关键流程

### 7.1 启动

```text
打开 App -> 进入首页
```

### 7.2 发布笔记

```text
首页点击发布 -> 输入文字或添加图片 -> 发布
  -> 选择拍照或从相册多选图片
  -> 保存笔记
  -> 调用 LLM 或备用模板生成评论
  -> 回到首页
  -> 评论按短延迟出现
```

### 7.3 打开我的页

```text
首页点击头像 -> 我的页 -> AI 好友页 / UI 实验室 / 关于页
```

### 7.4 查看与互动

```text
首页点击笔记卡片 -> 笔记详情
  -> 点击喜欢，原地切换喜欢状态
  -> 点击评论喜欢，原地切换评论喜欢状态
  -> 点击回复，弹出回复输入层
  -> 发送回复，目标评论下方出现本地二级回复
  -> 删除自己发送的本地二级回复
```

## 八、验收标准

- 用户打开 App 后直接进入首页，并可以发布第一条笔记。
- 首页没有底部导航。
- 用户可以从首页头像进入我的页。
- 用户可以从首页发布按钮进入发布笔记页。
- 用户可以发布纯文字笔记。
- 用户可以发布纯图片笔记。
- 用户可以发布文字 + 图片笔记。
- 发布页点击添加图片后可以选择拍照或相册。
- 相册选择支持一次添加多张图片，且图片总数最多 9 张。
- 已选相册图片再次打开相册时保持勾选，取消勾选后会从发布页移除。
- 首页可以看到已发布笔记。
- 笔记详情页可以看到 AI 点赞数和 AI 评论。
- 笔记详情页可以切换帖子喜欢状态。
- 笔记详情页可以切换评论喜欢状态。
- 笔记详情页点击回复后出现回复输入层。
- 笔记详情页发送回复后，目标评论下方出现本地二级回复。
- 笔记详情页可以删除自己发送的本地二级回复。
- 发布后 10 秒内至少出现 1 条 AI 评论。
- 至少 5 个 AI 好友具有可区分的名字、头像、关系和说话风格。
- LLM 调用失败时，帖子不会丢失。
