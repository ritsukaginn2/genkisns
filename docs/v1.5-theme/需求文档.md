## 一、版本目标

V1.5 在 V1 核心闭环跑通后，把 UI 实验室升级为主题设置。用户可以在 A/B/C 三套 UI 主题之间切换，切换后整个 App 的视觉风格同步变化，但功能、数据、页面跳转和交互保持一致。

---

## 二、功能范围

### 2.1 主题选择

- 在我的页提供主题设置入口。
- 主题设置页展示 A/B/C 三套主题。
- 用户选择某一套主题后，整个 App 立即应用该主题。
- 当前选中的主题需要有明确选中状态。

### 2.2 三套完整主题

- A 主题：完整实现首页、发布页、笔记详情页、我的页、AI 好友页、关于页、主题设置页和必要弹层。
- B 主题：沿用 V1 正式 UI。
- C 主题：完整实现首页、发布页、笔记详情页、我的页、AI 好友页、关于页、主题设置页和必要弹层。
- 三套主题必须支持相同功能、相同数据、相同导航和相同交互。

### 2.3 主题持久化

- 用户选择的主题在 App 重启后保持。
- 默认主题为 B。

## 三、页面清单

| 页面 | 内容 |
|------|------|
| 主题设置页 | A/B/C 主题选择、当前主题状态 |
| A 主题完整页面组 | V1 全部页面和弹层 |
| B 主题完整页面组 | V1 全部页面和弹层 |
| C 主题完整页面组 | V1 全部页面和弹层 |

---

## 四、验收标准

- 用户可以从我的页进入主题设置页。
- 用户可以在 A/B/C 三套主题之间切换。
- 切换主题后首页、发布页、笔记详情页、我的页、AI 好友页、关于页和弹层均使用新主题。
- 切换主题不影响已发布笔记、AI 评论、喜欢状态和本地回复。
- 三套主题下 V1 主流程都可以通过 `flutter run` 验收。
