## 一、版本目标

V1.6 把 V1 的本地模板 AI 互动升级为真实 LLM 生成，同时新增后端审核能力，避免 App 直接持有模型供应商密钥，也避免未经审核的请求直接进入 LLM。

V1.6 仍然不是完整云端 SNS 后端。用户笔记、图片、视频、喜欢状态和本地回复继续以手机本地 SQLite 与本地文件为主；后端只处理审核、风控、任务队列和 LLM 调用结果。

当前代码状态：

- Flutter App 已通过 `LLMClient` 接入可配置的 V1.6 后端地址。
- `services/llm-proxy` 已实现审核、限流、预算保护、队列化 LLM 代理、内部审核状态接口和测试。
- 发布后成功返回的真实 LLM 结果会写回本地 SQLite；失败、受限、审核不通过或结果非法时继续保留本地模板互动。
- 旧的“付费额度优先”方案不再作为 V1.6 文档主线；额度、订阅和购买验证可以作为后续商业化能力另行拆分。

## 二、功能范围

### 2.1 匿名安装身份

- App 首次启动时获取或生成本机 `installation_id`。
- `installation_id` 保存在本机。
- App 调用后端时携带 `installation_id`。
- V1.6 不提供手机号、邮箱、Apple 登录或 Google 登录。
- 后端根据 `installation_id` 维护审核状态、限流状态和必要的风控标记。

### 2.2 用户/安装实例审核

- 后端保存每个 `installation_id` 的状态：`allowed`、`limited`、`blocked`。
- `allowed` 可以正常请求 LLM 生成。
- `limited` 可以进入更严格的频控或只返回本地 fallback 指令。
- `blocked` 不再创建 LLM 任务，App 保持本地模板评论。
- 后端需要记录状态变更原因、时间和操作者来源；V1.6 可以先支持后台脚本或内部接口，不要求完整管理后台。

### 2.3 内容安全审核

- App 发起真实 LLM 生成前，把必要文本信息发给后端。
- 后端先对发帖文字、用户昵称、用户简介和 AI 好友人设做安全审核。
- V1.6 不把图片或视频文件发送给后端；只发送媒体类型、图片数量、视频存在与否等元数据。
- 审核不通过时，后端不调用 LLM，返回可 fallback 的状态。
- 审核异常或超时时，App 仍允许发布笔记，并保留本地模板评论。

### 2.4 LLM 代理后端

- 后端服务目录为 `services/llm-proxy`。
- App 不直接调用 OpenAI、Claude、Gemini 或其他 LLM 供应商。
- App 不保存或展示托管 LLM 供应商密钥。
- 后端通过云端 secret 或环境变量保存真实供应商密钥。
- 后端优先支持 OpenAI-compatible 供应商。
- 后端负责 prompt 拼装、模型调用、结构化 JSON 解析和结果校验。

### 2.5 Queue-first 生成

- App 发起生成请求后，后端创建生成任务。
- 后端先检查安装实例审核状态、内容审核结果、限流和预算保护。
- 通过检查后，任务进入队列。
- Worker 按固定并发消费队列并调用 LLM。
- App 通过轮询获取任务结果。
- 任务超时、失败、审核不通过或后端繁忙时，App 使用本地模板评论。

### 2.6 限流与预算保护

- 后端限制单个 `installation_id` 的请求频率。
- 后端限制单个 IP 的请求频率。
- 后端设置每日全局生成上限。
- 后端设置每日全局预算上限。
- 达到上限时，后端返回需要本地 fallback 的状态，不阻塞用户发布笔记。

### 2.7 LLM 结果

- LLM 只接收发帖文字、媒体元数据、用户昵称、用户简介和 AI 好友人设。
- LLM 返回结构化 JSON。
- 后端校验 JSON 合法性、评论数量、角色 ID 和字段范围。
- 后端返回 AI 点赞数、评论内容、评论喜欢数和任务状态。
- App 把成功结果写入本地 SQLite；失败时继续保留本地模板互动。

## 三、页面和 App 行为

| 页面 / 行为 | 内容 |
|-------------|------|
| 发布笔记 | 先本地保存和本地模板评论，再后台尝试真实 LLM 结果 |
| 审核受限提示 | 当后端明确返回 `limited` 或 `blocked` 时，可提示本次使用本地模板互动 |
| 关于页 | 说明 V1 本地生成；V1.6 接入后，文本会经后端审核后用于 LLM 生成 |

V1.6 不要求新增完整账号页、管理后台、订阅页或支付页。现有移动端“权益和额度”相关代码属于历史预留，后续实现前需要按新的后端范围决定保留、重命名或移除。

## 四、后端接口

| 接口 | 用途 |
|------|------|
| `POST /v1/installations` | 注册或刷新匿名安装身份 |
| `GET /v1/installations/me` | 查询当前安装实例审核状态和后端可用状态 |
| `POST /v1/interactions/jobs` | 提交内容审核和 AI 互动生成任务 |
| `GET /v1/interactions/jobs/:job_id` | 查询生成任务结果 |

后台审核操作可以先通过内部脚本、数据库操作或受保护的内部接口完成；不进入移动端公开 API。

## 五、关键流程

### 5.1 启动

```text
打开 App
  -> 读取或生成 installation_id
  -> 后台注册或刷新安装实例
  -> 查询审核/可用状态
  -> 进入首页
```

### 5.2 发布并生成 AI 互动

```text
发布笔记
  -> 本地保存笔记
  -> 本地模板生成点赞和评论
  -> 后台创建 LLM 生成任务
  -> 后端检查安装实例审核状态
  -> 后端审核文本内容
  -> 后端检查限流和预算保护
  -> 入队
  -> Worker 调用 LLM
  -> App 轮询任务结果
  -> 成功后替换为真实 LLM 互动并写入本地 SQLite
  -> 失败或受限时继续保留本地模板互动
```

### 5.3 审核受限

```text
发布笔记
  -> 本地保存笔记
  -> 后端返回 limited / blocked / content_rejected
  -> App 保留本地模板评论
  -> 必要时展示轻量提示
```

## 六、验收标准

- 后端可以注册或刷新 `installation_id`。
- 后端可以返回安装实例审核状态：`allowed`、`limited`、`blocked`。
- 后端可以拒绝 blocked 安装实例的 LLM 任务，且 App 不丢失本地发布内容。
- 后端可以在调用 LLM 前完成文本内容安全审核。
- 审核不通过时，后端不调用 LLM，并返回 App 可 fallback 的状态。
- 后端任务通过队列调用 LLM，不直接同步无限转发请求。
- App 可以轮询任务结果并把成功结果写入本地 SQLite。
- LLM 返回异常、超时或 JSON 不合法时，App 使用本地模板评论。
- 后端可以限制单个 `installation_id` 的请求频率。
- 后端可以限制单个 IP 的请求频率。
- 后端可以执行每日全局生成上限和预算保护。
- App 内不保存托管 LLM 供应商密钥。
- 图片和视频文件不会在 V1.6 发送给后端或 LLM。
