## 一、版本目标

V1.6 把 V1 的本地模板 AI 互动升级为真实 LLM 生成，同时建立付费额度、限流、队列和熔断，避免所有模型调用成本由开发者无限承担。

V1.6 仍然不是完整云端 SNS 后端。用户笔记、图片、喜欢状态和本地回复继续保存在手机本地。

---

## 二、功能范围

### 2.1 匿名安装身份

- App 首次启动时生成本机 `installation_id`。
- `installation_id` 保存在本机。
- App 调用 LLM 后端时携带 `installation_id`。
- V1.6 不提供手机号、邮箱、Apple 登录或 Google 登录。

### 2.2 付费与额度

- 提供免费额度。
- 提供 Pro 订阅权益。
- 免费额度和 Pro 额度都按生成次数计算。
- 每次成功生成一组 AI 互动扣除 1 次额度。
- LLM 后端失败、超时或返回不可用结果时不扣额度。
- 额度不足时，App 仍允许发布笔记，但使用本地模板评论。

### 2.3 订阅购买

- iOS 使用 Apple In-App Purchase。
- Android 使用 Google Play Billing。
- App 展示当前权益状态、剩余额度和订阅入口。
- 后端验证购买凭证后更新权益。
- V1.6 不支持网页支付。

### 2.4 LLM 后端

- 新增 `services/llm-proxy` 后端服务。
- App 不直接调用 OpenAI、Claude、Gemini 或其他 LLM 供应商。
- App 不保存或展示 LLM 供应商密钥。
- 后端通过云端 secret 或环境变量保存真实供应商密钥。
- 后端支持 OpenAI-compatible 供应商。

### 2.5 Queue-first 生成

- App 发起生成请求后，后端创建生成任务。
- 后端在额度通过后把任务放入队列。
- Worker 按固定并发消费队列并调用 LLM。
- App 通过轮询获取任务结果。
- 任务超时、失败或后端繁忙时，App 使用本地模板评论。

### 2.6 限流与熔断

- 后端限制单个 `installation_id` 的请求频率。
- 后端限制单个 IP 的请求频率。
- 后端设置每日全局生成上限。
- 后端设置每日全局预算上限。
- 达到上限时，后端直接返回需要本地 fallback 的状态。

### 2.7 LLM 结果

- LLM 只接收发帖文字、图片数量、用户昵称、用户简介和 AI 好友人设。
- V1.6 不把图片文件发送给 LLM。
- LLM 返回结构化 JSON。
- 后端校验 JSON 合法性、评论数量、角色 ID 和字段范围。
- App 把成功结果写入本地 SQLite。

## 三、页面清单

| 页面 | 内容 |
|------|------|
| 权益与额度页 | 当前套餐、剩余额度、订阅入口 |
| 订阅页 | Pro 权益、价格、购买按钮、恢复购买 |

## 四、后端接口

| 接口 | 用途 |
|------|------|
| `POST /v1/installations` | 注册或刷新匿名安装身份 |
| `GET /v1/entitlements` | 查询当前权益和剩余额度 |
| `POST /v1/purchases/verify` | 验证 Apple / Google 购买凭证 |
| `POST /v1/interactions/jobs` | 创建 AI 互动生成任务 |
| `GET /v1/interactions/jobs/:job_id` | 查询生成任务结果 |

## 五、关键流程

### 5.1 启动

```text
打开 App
  -> 读取或生成 installation_id
  -> 查询权益和剩余额度
  -> 进入首页
```

### 5.2 发布并生成 AI 互动

```text
发布笔记
  -> 本地保存笔记
  -> 创建 AI 互动生成任务
  -> 后端检查额度和限流
  -> 入队
  -> Worker 调用 LLM
  -> App 轮询任务结果
  -> 成功后写入本地 SQLite
```

### 5.3 额度不足

```text
发布笔记
  -> 后端返回额度不足
  -> App 使用本地模板评论
  -> 展示订阅入口
```

### 5.4 购买订阅

```text
进入订阅页
  -> 用户购买 Pro
  -> App 获取购买凭证
  -> 后端验证凭证
  -> 更新权益和额度
  -> App 刷新权益状态
```

## 六、验收标准

- App 可以生成并持久保存 `installation_id`。
- App 可以查询权益和剩余额度。
- 额度充足时，发布笔记会创建 LLM 生成任务。
- 后端任务通过队列调用 LLM，不直接同步无限转发请求。
- App 可以轮询任务结果并写入本地 SQLite。
- LLM 返回异常时，App 使用本地模板评论，且不扣额度。
- 额度不足时，App 仍可发布笔记，并使用本地模板评论。
- 额度不足时，App 能展示订阅入口。
- iOS 可以完成订阅购买和恢复购买。
- Android 可以完成订阅购买和恢复购买。
- 后端可以验证 Apple / Google 购买凭证。
- 后端可以限制单个 `installation_id` 的请求频率。
- 后端可以限制单个 IP 的请求频率。
- 后端可以执行每日全局生成上限和预算熔断。
- App 内不保存真实 LLM 供应商密钥。
- 图片文件不会发送给 LLM。
